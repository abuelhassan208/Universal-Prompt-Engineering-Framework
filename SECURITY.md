## ğŸš¨ Reporting a Vulnerability

If you discover a security vulnerability in the Universal Prompt Engineering Framework, please report it responsibly.

### How to Report

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, please:

1. **Email**: Send details to **aboualvixx@gmail.com**
2. **Subject Line**: Use "SECURITY: [Brief Description]"
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
   - Your contact information

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 5 business days
- **Status Updates**: Every 7 days until resolved
- **Resolution**: Depends on severity (see timeline below)

### Response Timeline

| Severity | Response Time | Fix Timeline |
|----------|--------------|--------------|
| Critical | 24 hours | 7 days |
| High | 48 hours | 14 days |
| Medium | 5 days | 30 days |
| Low | 7 days | 60 days |

## ğŸš« Known Limitations

### What This Framework Does NOT Do

- âŒ **No Data Storage**: Does not store user data
- âŒ **No External Calls**: No API calls to external services
- âŒ **No User Tracking**: No analytics or tracking
- âŒ **No Authentication**: No user accounts or login

### Potential Risks

1. **AI Model Limitations**
   - Framework quality depends on underlying AI model

   - Results may vary across different models

   - Always review generated prompts
